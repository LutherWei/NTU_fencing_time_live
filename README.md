## 🎯 系統簡介

本系統為台大擊劍隊量身打造的網頁式計分平台，支援：
- **公開瀏覽**：所有人都可以即時查看比賽分數與進度
- **管理員控制**：只有登入的管理員可以編輯分數、管理組別與選手
- **多組別獨立運作**：不同組別（如男子鈍劍、女子軍刀等）可同時進行，互不干擾

## ✨ 功能特色

### 1. 選手檢錄與組別管理
- 自由新增/編輯/刪除組別
- 每個組別獨立管理參賽選手
- 選手檢錄狀態追蹤

### 2. 分組預賽
- 自動生成對戰矩陣表（4-8人一組）
- 即時計分並自動計算：
  - 勝場數與勝率
  - 得分(TS)、失分(TR)、淨得失分(Ind)
- 智慧排名系統（依勝率 → 淨得分 → 總得分）

### 3. 淘汰賽
- 自訂淘汰比率
- 自動生成樹狀圖（2^n 強制）
- 最大亂度排程（強弱對決）
- 支援三四名決定戰
- 輪空自動處理

### 4. 即時結果展示
- 公開的比賽結果頁面
- 樹狀圖視覺化呈現
- 自動更新總排名

## 🚀 快速開始

### 1. 環境需求
- Node.js 20 或以上版本
- npm 或 yarn

### 2. 安裝步驟

```bash
# 進入專案目錄
git clone git@github.com:LutherWei/NTU_fencing_time_live.git
cd fencing-score-system

# 安裝相依套件
npm install

# 設定環境變數
cp .env.example .env
# 編輯 .env 設定資料庫連線和 JWT Secret

# 初始化資料庫
npx prisma generate
npx prisma db push

# (可選) 建立初始管理員帳號
npx prisma db seed
```

### 3. 啟動測試伺服器

```bash
npm run dev
```
開啟瀏覽器前往 [http://localhost:3000]


### 4. 建置正式環境

```bash
tmux
npm run build
npm run start -- -p 3001
```
For ngrok
```bash
tmux
./ngrok http --domain=rolanda-indistinguishable-agonisedly.ngrok-free.dev 3001
```
開啟瀏覽器前往 [https://rolanda-indistinguishable-agonisedly.ngrok-free.dev/]

## 📖 使用指南

### 一般使用者

#### 查看即時賽況

1. **首頁** (`/`)
   - 顯示所有進行中的組別
   - 查看各組別當前狀態（檢錄中/分組賽/淘汰賽/已完賽）
   - 快速連結到各組別詳細結果

2. **組別結果頁** (`/results/[categoryId]`)
   - 查看該組別完整資訊
   - 分組預賽成績表與統計
   - 淘汰賽樹狀圖
   - 最終排名

### 管理員操作

#### 步驟 1：登入系統

1. 前往 **管理員登入** (`/admin/login`)
2. 輸入帳號和密碼
3. 登入成功後會自動導向管理員儀表板

**預設帳號**（如有執行 seed）：
- 帳號：`admin`
- 密碼：`admin123`（請務必修改！）

#### 步驟 2：選手檢錄與組別管理

1. 前往 **選手檢錄** (`/admin/check-in`)

2. **新增組別**
   - 點擊「新增組別」按鈕
   - 輸入組別名稱（例如：男子鈍劍、女子軍刀）
   - 確認新增

3. **管理選手**
   - 選擇組別
   - 點擊「新增選手」輸入選手姓名
   - 勾選確認選手已完成檢錄
   - 可編輯或刪除選手資料

4. **進入下一階段**
   - 確認所有選手檢錄完畢後
   - 點擊「開始分組賽」進入分組編排

#### 步驟 3：分組預賽管理

1. 前往 **分組預賽** (`/admin/poules/[categoryId]`)

2. **分組編排**
   - 手動將選手分配到各小組（A組、B組...）
   - 每組限制 4-8 人
   - 確認分組後系統自動生成對戰矩陣

3. **登記分數**
   - 對戰矩陣說明：
     - `form[i][j]` = 第 i 位選手對第 j 位選手的得分
     - 對角線（自己vs自己）會顯示為 X 或黑色格子
     - 每場比賽打到 5 分
   - 輸入分數：
     - 如果比分為 5:3，勝者格填入 `V`，敗者格填入 `3`
     - 如果比分為 5:4，勝者格填入 `V`，敗者格填入 `4`
   - 顏色標示：
     - ✅ 綠色 = 勝利
     - ❌ 紅色 = 輸家

4. **自動計算統計**
   系統會即時計算：
   - **勝場數** (V)
   - **勝率** (V/M)
   - **得分** (TS = 該列總和)
   - **失分** (TR = 該欄總和)
   - **淨得失分** (Ind = TS - TR)

5. **儲存與排名**
   - 每格輸入完自動儲存
   - 所有小組完成後，系統自動計算種子排名
   - 排名規則：勝率 → 淨得分 → 總得分
   - 若三項完全相同且涉及晉級資格：一律晉級

#### 步驟 4：淘汰賽管理

1. 前往 **淘汰賽** (`/admin/bracket/[categoryId]`)

2. **設定淘汰參數**
   - **淘汰比率**：設定晉級淘汰賽的選手比例
     - 例如：20% 表示前 20% 選手進入淘汰賽
   - **三四名決賽**：勾選是否要進行三四名決定戰

3. **樹狀圖生成**
   - 確認參數後系統自動生成淘汰賽樹狀圖
   - 採用單淘汰制
   - 若晉級人數不是$2^n$，會安排輪空 (Bye)
   - 對戰組合採「最大亂度」原則：
     - 第 1 名 vs 最後一名
     - 第 2 名 vs 倒數第二名
     - 依此類推
     - 排列方式比照FIE系統

4. **登記淘汰賽分數**
   - 點擊樹狀圖上的任一場比賽
   - 輸入兩位選手的得分
   - 確認後系統自動：
     - 判定勝者
     - 更新下一輪對戰表
     - 若有排名互換需求會自動處理

5. **查看總排名**
   - 淘汰賽結束後自動生成最終排名
   - 依淘汰賽結果決定名次
   - 公開結果可供所有人查看

#### 其他管理功能

- **管理員儀表板** (`/admin/dashboard`)
  - 總覽所有組別狀態
  - 快速連結到各功能頁面
  - 查看系統統計資訊

- **登出**
  - 點擊導覽列的「登出」按鈕
  - 或前往 `/api/auth/logout`

## 🗄 資料庫結構

### 主要資料表

| 資料表 | 說明 |
|--------|------|
| `Admin` | 管理員帳號資料 |
| `Category` | 比賽組別（組別名稱、進度狀態） |
| `Fencer` | 選手資料（姓名、檢錄、統計數據、排名） |
| `Poule` | 分組預賽小組 |
| `PouleMatch` | 分組預賽單場比賽 |
| `Bracket` | 淘汰賽設定 |
| `EliminationMatch` | 淘汰賽單場比賽 |

詳細結構請參考 `prisma/schema.prisma`

## 🔌 API 接口

### 認證相關
- `POST /api/auth/login` - 管理員登入
- `POST /api/auth/logout` - 管理員登出
- `GET /api/auth/me` - 取得當前管理員資訊

### 組別管理
- `GET /api/categories` - 取得所有組別
- `POST /api/categories` - 新增組別
- `GET /api/categories/[categoryId]` - 取得單一組別詳細資料
- `PATCH /api/categories/[categoryId]` - 更新組別資料
- `DELETE /api/categories/[categoryId]` - 刪除組別

### 選手管理
- `GET /api/fencers` - 取得選手清單
- `POST /api/fencers` - 新增選手
- `PATCH /api/fencers/[fencerId]` - 更新選手資料
- `DELETE /api/fencers/[fencerId]` - 刪除選手

### 分組預賽
- `GET /api/poules` - 取得所有小組
- `POST /api/poules` - 建立小組
- `GET /api/poules/[pouleId]/matches` - 取得小組所有對戰
- `PATCH /api/poules/[pouleId]/matches` - 更新小組比賽分數

### 淘汰賽
- `GET /api/bracket?categoryId=[id]` - 取得淘汰賽樹狀圖
- `POST /api/bracket` - 建立淘汰賽
- `PATCH /api/bracket/[matchId]` - 更新淘汰賽比賽分數
